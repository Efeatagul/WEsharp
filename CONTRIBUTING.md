# Contributing to WSharp

We welcome technical contributions that align with the scientific and performance goals of the WSharp project. To maintain the integrity of the JIT compiler and simulation engine, please follow these guidelines:

## How to Contribute
1. **Track Issues:** Monitor the "Issues" tab to identify current priorities, bug reports, and requested features.
2. **Technical Support:** You can contribute by providing solutions to active issues or proposing architectural enhancements.
3. **Testing Requirements:** All code contributions must successfully pass the existing test suite located in the `/tests` directory. Stability is mandatory.
4. **Discussions:** For significant changes or new module proposals, please open a new issue first to discuss the implementation strategy with the core maintainers.

## Priorities
Our primary focus is the accuracy of biophysical simulations and the optimization of JIT compilation throughput. Contributions that improve Big O complexity or reduce memory latency are highly encouraged.
